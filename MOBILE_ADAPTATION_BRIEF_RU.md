# Logist City — техбриф для мобильной адаптации

## 1) Репозиторий и деплой
- GitHub: [https://github.com/timur-ship-it/logist-city](https://github.com/timur-ship-it/logist-city)
- Ветка: `main`
- Публичный сайт: [https://timur-ship-it.github.io/logist-city/](https://timur-ship-it.github.io/logist-city/)
- Локальный клон: `/tmp/netlify-deploy`

## 2) Стек и архитектура
- Проект без сборки: один статический файл `[index.html](/tmp/netlify-deploy/index.html)` (HTML + CSS + JS внутри).
- Ассеты товаров: папка `[images](/tmp/netlify-deploy/images)`.
- Служебные скрипты генерации/пересборки данных: `[scripts](/tmp/netlify-deploy/scripts)`.
- Состояние формы хранится в браузере: `localStorage` ключ `konex_procurement_v1`.

## 3) Текущая структура каталога (в `index.html`)
Категории (`.cat-section`):
- `abrasives`
- `chemicals`
- `tapes`
- `ppe`
- `fasteners`
- `electrical`
- `rags`
- `geo`

Карточка товара (`.product`) содержит:
- `data-id` (внутренний ID)
- `data-search` (поисковый индекс)
- иногда `data-form-profile` (для абразивов/APC)
- при скрытии позиции: `data-removed="true"`

Фактические числа в текущей версии:
- Всего карточек в HTML: `135`
- Скрытых (`data-removed="true"`): `2` (`id=20,21`)
- Активных к отображению: `133`
- Дополнительные APC-позиции: `id 221..249` (29 карточек)

## 4) Ключевая логика JS (важно не сломать)
Основные функции (в конце `index.html`):
- `saveState()` / `loadState()` / `clearAll()` — сохранение/восстановление формы
- `collectOrderLines()` — сбор выбранных позиций для экспорта/отправки
- `exportCSV()` — выгрузка CSV
- `submitForm()` + `buildPayload()` — отправка в Google Apps Script
- `initProductMetadata()` — автоприсвоение артикулов вида `ABR-001`, `CHM-001`, ...
- `initAbrasiveSizeSelectors()` — селект размеров/параметров для абразивов
- `renderAbrasiveShapeBadges()` — бейджи типа профиля (`T41`, `T42`, `T27`, `T29`)

Константы/мапы, которые завязаны на поведение:
- `STORAGE_KEY`
- `CAT_SKU_PREFIX`
- `ABRASIVE_SIZE_OPTIONS`
- `ABRASIVE_DELTA_SIZE_OPTIONS`
- `ABRASIVE_SHAPE_MAP`
- `GOOGLE_SCRIPT_URL`

## 5) Внешняя интеграция (submission)
Текущий endpoint:
- `GOOGLE_SCRIPT_URL = https://script.google.com/macros/s/AKfycbxyjqXGolhSOAFRdVZFIfDJ6G-xN54VfbrjrueFFp66HGstF3BsAgpvnv8rTRfBiJ2w4A/exec`

Формат отправки:
- JSON payload с `company`, `period`, `comments`, `products[]`
- В `products[]` передаются `id`, `sku/article`, `name`, `qty`, `unit`, `size`, `params`, `note` и пр.

Важно: при mobile-рефакторинге структура payload и названия полей должны остаться совместимыми.

## 6) Что сейчас плохо на мобильных
Основные проблемы по текущей верстке:
- Карточка товара изначально горизонтальная (`.prod-img` + `.prod-body`), на узких экранах плотная.
- Блок вариантов размера `.size-variant-row` имеет 5-колоночную сетку и большие `minmax`, часто приводит к визуальной перегрузке.
- Много полей в одну строку (`qty`, `size`, `params`, `note`) неудобны для ввода с телефона.
- FAB-панель действий внизу может конфликтовать с клавиатурой и занимать полезную высоту экрана.
- Дропдауны размеров у абразивов длинные; на мобильном нужен более удобный паттерн выбора.

## 7) Техническое задание на mobile adaptation (для другого чата)
Сделать полноценную мобильную адаптацию без изменения бизнес-логики:

1. Сохранить все существующие функции JS и структуру данных отправки.
2. Перестроить UI под `<= 768px`, дополнительно оптимизировать для `<= 430px`.
3. Карточку товара сделать вертикальной:
- картинка сверху
- название/описание
- затем поля ввода блоками
4. Для абразивов:
- селект `Размер` на всю ширину
- `Параметры` и `Комментарий` отдельными строками
- строки `+ Добавить размер` рендерить как вертикальные мини-карточки, а не 5-колоночную строку
5. Увеличить touch-area интерактивных элементов до удобного мобильного размера.
6. Убрать горизонтальный скролл страницы полностью.
7. FAB-панель сделать мобильной:
- компактнее
- не перекрывает активный input (учесть экранную клавиатуру)
8. Сохранить визуальную иерархию категорий и быстрый поиск.

## 8) Acceptance criteria
- На iPhone шириной 390px нет горизонтального скролла.
- Можно выбрать товар, указать несколько размеров одной позиции и количества.
- Размер/параметры/примечание корректно попадают в:
- `collectOrderLines()`
- `exportCSV()`
- `submitForm()` payload
- Артикулы (`ABR-...`, `CHM-...` и т.д.) не ломаются.
- `localStorage` восстановление после перезагрузки работает как раньше.
